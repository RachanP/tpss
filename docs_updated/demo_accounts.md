# บัญชี Demo + วิธีเตรียมข้อมูลให้ลูกค้าทดสอบ

> ใช้สำหรับ demo / ให้ลูกค้าเข้าทดสอบเอง — branch `feat/v2-seeder-demo`

## วิธีเตรียมข้อมูล (1 ครั้ง)

```bash
php artisan migrate:fresh --seed
php artisan db:seed --class=ClientDemoSeeder
```

- `migrate:fresh --seed` = master data + บัญชีผู้ใช้ (สถานะ `preparation` — ยังไม่เปิดจัดตาราง)
- `ClientDemoSeeder` = เปิดปีการศึกษา 2569 (scheduling) + สร้าง offering + ตารางสาธิตแบบผสมสถานะ + ตัวชน

> ⚠️ `ClientDemoSeeder` **ไม่ถูกเรียกใน `DatabaseSeeder`** โดยตั้งใจ — เพื่อให้ Playwright E2E ยังเริ่มจาก baseline เดิม (`preparation`) ได้ ต้องสั่งรันแยกเอง

### การชนที่ seed ไว้ใน NSBS 212 (login `head_med` → ตารางสอน / หน้าแจ้งเตือน)

> ⚠️ seed เฉพาะการชน **"ข้ามวิชา"** เท่านั้น — การชน **"ในวิชา"** สร้างไม่ได้จริง เพราะ realtime check
> บล็อกปุ่มบันทึกตั้งแต่ตอนกรอก modal (จะไม่มีวันมีในข้อมูลจริง) · การ์ด "การชนข้ามวิชา" จึงมีไว้แสดงเฉพาะการชนข้ามรายวิชา

| เคส | กิจกรรม | ชนกับ | ประเภท |
|-----|---------|-------|--------|
| [1] ชนห้อง **ข้าม-วิชา** | จ. 09–12 (ห้อง R-302) | NSBS 111 (ห้องเดียวกัน เวลาเดียวกัน) | `room_overlap` |
| [2] ชนผู้สอน **ข้าม-วิชา** | อ. 13–16 (instructor_01) | NSBS 111 (ผู้สอนคนเดียวกัน เวลาเดียวกัน) | `instructor_overlap` |
| [3] **ห้องใช้ร่วม = ไม่ชน** | ศ. 13–16 ห้อง WARD-A (หอผู้ป่วย) | NSBS 111 (ห้อง+เวลาเดียวกัน แต่เป็นห้องใช้ร่วม) | **ไม่แจ้งชน** |

> **เคส [3]** สาธิตว่า สถานที่ฝึกที่ตั้ง `is_shared=true` (หอผู้ป่วย/โรงพยาบาล/ชุมชน ฯลฯ) หลายวิชาใช้พร้อมกันได้ → ระบบไม่ถือว่าชน
> การชนข้ามวิชารวม 4 slot (ห้อง 2 + ผู้สอน 2 ฝั่ง NSBS 111/212) → การ์ด "การชนข้ามวิชา" = 4

## ข้อมูลเพิ่มเติมที่ใช้ประกอบการทดสอบ

### ไอดีผู้ใช้ (วิธีค้นหา)
ถ้าต้องการไอดีผู้ใช้จริงในฐานข้อมูล ให้รันคำสั่งต่อไปนี้บนเครื่องที่รัน Laravel (หลังรัน seeder แล้ว):

```bash
# ดู id ของบัญชี demo ที่เราสร้างไว้
php artisan tinker --execute "echo json_encode(\App\Models\User::whereIn('username', ['admin_01','head_med','head_psy','exec_01','instructor_01','staff_01'])->get()->pluck('id','username')->toArray());"

# หรือใช้ SQL ถ้ามี client mysql:
mysql -u<user> -p -h<host> -P<port> -e "SELECT id,username,name FROM users WHERE username IN ('admin_01','head_med','head_psy','exec_01','instructor_01','staff_01');" tpss
```

(ผลลัพธ์จะแสดง mapping `username => id` ซึ่งใช้เมื่ออ้างอิงผู้ใช้ในสคริปต์ทดสอบหรือเมนูผู้ดูแล)

### รายละเอียดเคสการชน / warning (จาก `ClientDemoSeeder`)
ต่อไปนี้เป็นรายละเอียดเชิงเทคนิคของแต่ละเคสที่ seed ไว้ — ใช้ประกอบการอธิบายหน้าจอแจ้งเตือนและเพื่อทำ reproduce ใน presentation

- NSBS 111 (ต้นทาง — สถานะ `approved` / `published`):
	- จันทร์: 09:00-12:00, ห้อง = `roomB` (ตัวอย่าง: R-302), ผู้สอน = `instructor_01` (อนุมัติ)
	- อังคาร: 13:00-16:00, ห้อง = `roomB`, ผู้สอน = `instructor_01` (อนุมัติ)
	- ศุกร์: 13:00-16:00, ห้อง = `roomShared` (is_shared=true), ผู้สอน = `instructor_01` (อนุมัติ)

- NSBS 212 (เคสโชว์การชน / warnings — สถานะ `draft`):
	- [1] Room overlap (ชนห้องข้ามวิชา): จันทร์ 09:00-12:00, ห้อง = `roomB` — **ชนกับ NSBS 111 จันทร์ 09:00-12:00** (คนละผู้สอน) → `room_overlap`
	- [2] Instructor overlap (ชนผู้สอนข้ามวิชา): อังคาร 13:00-16:00, ห้อง = `roomA`, ผู้สอน = `instructor_01` — **instructor_01 สอน NSBS 111 ในช่วงเวลาเดียวกัน** → `instructor_overlap`
	- [3] Shared room (ไม่ถือว่าชน): ศุกร์ 13:00-16:00, ห้อง = `roomShared` (is_shared=true) — มี NSBS 111 ใช้พื้นที่เดียวกันพร้อมกัน → **ไม่แจ้งชน**
	- [W1] Incomplete (ข้อมูลไม่ครบ): ศุกร์ 09:00-12:00, ห้อง = `roomC`, **ไม่มีผู้สอน** → `incomplete`
	- [C1] Capacity test (ผูกกลุ่มนักศึกษา): ศุกร์ 14:00-16:00, ห้อง = `roomA`, ผู้สอน = `head_med` — สร้าง `StudentGroup` A1=25, A2=20 แล้วตั้ง `capacity_required=30` บน slot นี้ เพื่อเป็นตัวอย่าง `capacity_exceeded` หรือให้ทีมปรับความจุห้อง (ดู caveat ด้านล่าง)
	- [D1] Dept mismatch (ผู้สอนต่างภาควิชา): พฤหัสบดี 09:00-12:00, ห้อง = `roomC`, ผู้สอน = `head_psy` (มาจากภาควิชาสุขภาพจิต) → `dept_mismatch`
	- [R1] No lead (ไม่มีผู้สอนนำ): พฤหัสบดี 13:00-15:00, ห้อง = `roomB`, ผู้สอนมีแต่ `is_lead=false` → `no_role` / edge-case (UI อาจไม่โชว์เหมือน incomplete)
	- [H1] Holiday slot for `head_med`: พุธ (วันหยุด seed: 2026-06-03) 10:00-12:00, ห้อง = `roomA`, ผู้สอน = `head_med` → `holiday` warning

- NSBS 213 (สถานะ `pending_approval` — ไม่มีการชนใน seed):
	- พฤหัสบดี 13:00-16:00, ห้อง = `roomA`, ผู้สอน = lead of offering → `pending_approval`
	- ศุกร์ 09:00-12:00, ห้อง = `roomA`, ผู้สอน = lead → `pending_approval`

- NSBS 314 (สถานะ `draft`):
	- พุธ (2026-06-03) 09:00-12:00, ห้อง = `roomB`, ผู้สอน = lead → `holiday` warning (ตัวอย่างเพื่อโชว์ปฏิทินลงสีวันหยุด)

หมายเหตุเชิงปฏิบัติ:
- ชื่อห้อง `roomA/roomB/roomC/roomShared` ใน seeder คือตัวอย่าง: ในระบบจริงมีคอลัมน์ `rooms.code`/`rooms.name` — ตรวจสอบชื่อจริงจากหน้า `master data → ห้อง` ก่อนนำเสนอ
- `capacity_exceeded` จะเกิดขึ้นจริงเมื่อ `room.capacity` น้อยกว่าจำนวนทั้งหมดของ `student_groups` ที่ผูกกับ slot — seeder ผูกกลุ่มไว้แล้ว แต่ถ้าค่า `rooms.capacity` ของห้องที่ใช้ใน demo สูงพอ ระบบจะไม่แสดง `capacity_exceeded` จนกว่าจะปรับค่า `rooms.capacity` ให้ต่ำลง หรือเพิ่มขนาดกลุ่ม

ถ้าต้องการ ผมช่วยแก้ `ClientDemoSeeder` ให้บังคับเกิด `capacity_exceeded` จริง (โดยตั้ง `room.capacity` เป็นค่าน้อย เช่น 20 สำหรับ `roomA`) แล้วรัน seeder ให้ — ต้องการให้ผมทำไหม?

### warning อื่น ๆ ในหน้าแจ้งเตือน (ไม่ใช่ "การชน" — สีเหลือง)

หน้าแจ้งเตือนของหัวหน้าวิชา (`/maker/alerts`) รวม **การชน (สีแดง) + warning (สีเหลือง)** ไว้ที่เดียว · demo seed ตัวอย่าง warning ที่เกิดได้จริงในเฟสนี้:

| warning | seed ไว้ที่ | login ที่เห็น |
|---------|-----------|--------------|
| `incomplete` ข้อมูลไม่ครบ (ขาดผู้สอน) | NSBS 212 ศ. 09–12 | `head_med` |
| `holiday` กิจกรรมตรงวันหยุดราชการ | NSBS 314 พ. (วันเฉลิมพระชนมพรรษาฯ) | `head_psy` |

> warning ที่ระบบรองรับแต่ **ยังไม่ seed** (ไม่เกิดในเฟสปัจจุบัน):
> - `capacity_exceeded` (นศ.เกินความจุห้อง) — ต้องผูกกลุ่มนักศึกษา = Phase B (หลังอนุมัติ)
> - `no_role` (ผู้สอนไม่กำหนดบทบาท) · `dept_mismatch` (ผู้สอนต่างภาควิชา) — ปกติ UI กันไว้ตั้งแต่ตอนเพิ่มผู้สอน (department gate) จึงเป็น edge case

**ตัวเลขรวม:** ป้ายแจ้งเตือนบน sidebar (เมนู "แจ้งเตือน") = **การชน + warning ทุกชนิด** ตรงกับยอดรวมในหน้าแจ้งเตือน (คำนวณจากแหล่งเดียว `CoordinatorAlertService`) · เช่น `head_med` = 9 (ชน 8 + ข้อมูลไม่ครบ 1), `head_psy` = 1 (holiday)

**วันหยุดราชการ:** seed ไว้ 9 วัน (`source=manual`) ในช่วงปี 2569 → ปฏิทินลงสีวันหยุดได้ · ปกติระบบดึงจาก Google ICS อัตโนมัติตอน admin สร้าง/แก้ปีผ่าน UI (seeder ไม่ผ่าน flow นั้นจึง seed มือไว้)

> **group_overlap** (ชนกลุ่มนักศึกษา) ยังไม่ seed — V2 ย้ายการจัดกลุ่มนักศึกษาไปเฟส "หลังอนุมัติ" หัวหน้าวิชาจึงไม่ผูกกลุ่มกับ slot ในเฟสนี้

> **Master data — ห้องใช้ร่วม:** `LocationTypeSeeder` ตั้ง `is_shared=true` ให้ หอผู้ป่วย / โรงพยาบาล / ศูนย์พัฒนาเด็กฯ / โรงเรียน / ชุมชน / รพ.สต. (สถานที่ฝึกภายนอกขนาดใหญ่) ส่วนห้องเรียน/ห้องปฏิบัติการ/ห้องคลอด/ออนไลน์ = ไม่ใช้ร่วม

## Flow แนะนำให้ลูกค้าลอง

1. **admin_01** → ตั้งค่าระบบ → ดูปี 2569 เปิด scheduling อยู่ + master data ครบ
2. **head_med** → ตารางสอน → เลือก NSBS 212 → เห็นกิจกรรมวันจันทร์ขึ้นเตือนชนห้อง/ผู้สอน → หน้า "แจ้งเตือน" สรุปการชน
3. **head_psy** → NSBS 314 → ลองเพิ่มกิจกรรมใหม่เอง (ทดสอบ modal + การตรวจวันหยุด/สอบ)
4. **exec_01** → dashboard → เห็นจำนวนรายวิชาแยกสถานะ (published/pending/draft)
5. **instructor_01** → เห็นเฉพาะตารางที่ตัวเองสอน
